# Add favorites feature

## Goal
Let users mark animals as favorites, unfavorite them, and view their favorites list. Adds a `Favorite` resource to the API and a heart toggle on each animal in the web list (and detail page).

## Affected areas
- packages/api/Program.cs
- packages/api/Modules/Favorites/Favorite.cs
- packages/api/Modules/Favorites/FavoriteConfiguration.cs
- packages/api/Modules/Favorites/FavoriteContracts.cs
- packages/api/Modules/Favorites/FavoriteDbContext.cs
- packages/api/Modules/Favorites/FavoriteService.cs
- packages/api/Modules/Favorites/FavoriteEndpoints.cs
- packages/api/Migrations/ (new EF migration for the Favorites table)
- packages/web/src/api/schema.ts (regenerated, do not hand-edit)
- packages/web/src/api/favorites.ts (new client wrapper, mirrors `animals.ts`)
- packages/web/src/components/favorite-button.tsx (new heart-toggle component using `lucide-react`'s `Heart`)
- packages/web/src/routes/animals/index.tsx (add a column with the heart button per row)
- packages/web/src/routes/animals/$id.tsx (add the heart button next to the title)
- packages/web/src/routes/favorites/index.tsx (new route showing the user's favorite animals)

## Approach
1. Create `packages/api/Modules/Favorites/Favorite.cs` ‚Äî entity with `Id` (int), `AnimalId` (int FK), `CreatedAt` (DateTime). Skip `UserId` for now and call out the ambiguity under Open questions.
2. Add `FavoriteConfiguration.cs` ‚Äî `HasKey(Id)`, unique index on `AnimalId`, FK to `Animals` (cascade delete on animal removal).
3. Add `FavoriteDbContext.cs` mirroring `AnimalDbContext.cs`. (Alternative: extend `AnimalDbContext` with `DbSet<Favorite>`. Decide via Open questions.)
4. Add `FavoriteContracts.cs` ‚Äî `CreateFavoriteRequest { int AnimalId }` with `[Required]`/`[Range]` annotations matching the existing module's style.
5. Add `FavoriteService.cs` ‚Äî `GetAllAsync`, `CreateAsync(CreateFavoriteRequest)` (idempotent: return existing if already favorited), `DeleteAsync(int id)`.
6. Add `FavoriteEndpoints.cs` mirroring `AnimalEndpoints.cs`:
   - `GET /api/favorites` ‚Üí list.
   - `POST /api/favorites` (with `ValidationFilter<CreateFavoriteRequest>`) ‚Üí 201 Created.
   - `DELETE /api/favorites/{id:int}` ‚Üí 204 / 404.
7. Wire up in `Program.cs`: register `FavoriteDbContext` (or migrate to shared context), register `FavoriteService`, call `FavoriteEndpoints.Map(app)` after the existing `AnimalEndpoints.Map(app)`. Add `db.Database.EnsureCreated()` for the new context (matching how `AnimalDbContext` is initialized).
8. Generate an EF Core migration for the new table (piv-implement runs the dotnet ef tooling; do not run it here).
9. **Start the API on port 5204, then run `bun gen:api` in `packages/web`** to refresh `src/api/schema.ts`. The OpenAPI types must be regenerated against a running API or the web client cannot import the new schemas.
10. Add `packages/web/src/api/favorites.ts` mirroring `animals.ts`: `getFavorites`, `addFavorite(animalId)`, `removeFavorite(id)`. Export the generated DTO types.
11. Add `packages/web/src/components/favorite-button.tsx` ‚Äî a small client component taking `{ animalId, favoriteId | null, onToggle }`, renders `Heart` (filled when favorited), calls `addFavorite` / `removeFavorite`, optimistic update with `router.invalidate()` on success.
12. In `packages/web/src/routes/animals/index.tsx`, fetch favorites alongside animals in the loader, build a `Map<animalId, favoriteId>`, add a leading or trailing column whose cell renders `<FavoriteButton ‚Ä¶>`. Stop row-click propagation on the button.
13. In `packages/web/src/routes/animals/$id.tsx`, place a `FavoriteButton` next to the existing edit button.
14. Add `packages/web/src/routes/favorites/index.tsx` ‚Äî new route, loader fetches `getFavorites()` and renders a list of the favorited animals (resolve animal info either by joining server-side in the API response or by fetching animals and zipping client-side; defer the choice to Open questions).

## Out of scope
- Authentication / per-user scoping. This plan treats favorites as a single global set.
- Pagination or filtering of favorites.
- Animation/styling polish on the heart button beyond filled vs. outlined.
- Tests in `packages/api.tests` and `packages/api.tests.integration` (worth a follow-up plan).

## Open questions
- Should favorites be per-user? The repo has no auth today, so I'm assuming "global, single-user dev app." Confirm before piv-implement runs.
- Should the `Favorite` entity live in its own `FavoriteDbContext`, or be added to `AnimalDbContext` (so EF can model the FK natively)? The latter is simpler but couples modules.
- Should `GET /api/favorites` return `Favorite` rows (just ids + timestamps) or the joined `Animal` data? The web "favorites list" page wants animals, so returning a projection is more useful.
- "Item card" ‚Äî the current animals list is a `Table`, not cards. Confirm whether you want the heart on table rows (assumed) or a layout switch to cards.

- `dotnet test packages/api.tests` and `dotnet test packages/api.tests.integration` ‚Äî existing suites still green.
- Start the API (`dotnet run --project packages/api`), then in `packages/web` run `bun gen:api` and confirm `src/api/schema.ts` now contains `/api/favorites` paths and `CreateFavoriteRequest` schema. The API must be running on port 5204 for this to succeed.
- `bun check` in `packages/web` ‚Äî type-check + lint pass.
- Manual: load `/animals`, click the heart on a row, reload ‚Üí state persists. Click again ‚Üí unfavorited. Load `/favorites` ‚Üí favorited animal appears. Delete the underlying animal ‚Üí favorite cascades away.
